# Gemini API 連携に関する仕様と注意点

## 使用モデル
- モデル名: `gemini-3.5-flash`
- 理由: 2026年現在の最新Flashモデルであり、音声データの直接処理（マルチモーダル）に高速かつ安価に対応しているため。

## 実装上の重要ポイント
- **音声データの送信**: サーバーを介さないフロントエンド完結型とするため、File APIではなく、音声をBase64エンコードして送信する `inlineData` 形式を採用。
- **セキュリティ**: APIキーの漏洩を防ぐため、ソースコード（GitHub）への直書きを禁止。ブラウザの `localStorage` を利用した端末内保存を実装。
- **参照した公式ドキュメント**: https://ai.google.dev/gemini-api/docs/audio
